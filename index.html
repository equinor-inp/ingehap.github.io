      • pytest - homepage
        
  How to run pytest
  -----------------
  pytest                            looks in current folder and in subfolders for test files
                                    runs the test code it finds
  pytest --help                     prints help information
  pytest "directories"              as pytest, but with roots as given
  pytest "file names sep by space"  run all tests in the files
  pytest -v test_a.py::test_b       run function test_b located in file test_a
  pytest --version                  pytest lib version
  pytest --tb=no                    turns off stak trace
  pytest -q                         decrease verbosity
  pytest -l                         show locals in tracebacks
  pytest -x                         stops if a test fails
  pytest --collect-only             only collect tests, don't execute
  
  Naming conventions
  ------------------
  Default rules for making code discoverable by pytest are
  - name test files test_*.py or *_test.py
  - name test functions / modules test_*
  - name test classes Test*
  The default rules may be altered via a onfiguration file
  
  Understanding pytest output
  ---------------------------
  F = failures
  E = errors
  s = skips (defined by decorators @pytest.mark.skip() or @pytest.mark.skipif())
  x = xfails (expected to fail and fails; defined by @pytest.mark.xfail())
  X = xpasses (expected to fail, but passes)
  
  
  Configuration files
  -------------------
  pytest.ini        directives for changing behavior of pytest
  conftest.py       hook functions and fixtures (one in test, one in subdirs)
  toc.ini
  setup.cfg
  
  
  Compatibility with other test libraries
  ---------------------------------------
  pytest can run tests written for unittest and nose packages
  
  Marking test functions by decorators
  ------------------------------------
  @pytest.mark.simple
  def test_anything():
  ..
  >> pytest -v -m 'simple'
  
  Skipping test functions by decorators
  -------------------------------------
  @pytest.mark.skip(reason='bad code, must be fixed')
  def test_meaning_of_life():
  ..
  @pytest.mark.skipif(tasks.__version__ < '0.2.0')
  def test_other():
  >> pytest -rs
  
  Marking tests as expecting to fail
  ----------------------------------
  @pytest.mark.xfail(reason='missing input')
  def test_one():

      4.7.4 CS - Testing - Python Testing Library unittest

      • unittest - homepage
        
#angle.py
class Angle:
    def __init__(self, deg):
        self.deg = deg % 360
    def is_acute(self):
        return (self.deg < 90)
    def add(self, ang):
        return Angle(self.deg + ang.deg)

#test_angle.py
class TestAngle(unittest.TestCase):
    def test_deg(self):
        small = Angle(60)
        self.assertEqual(60, small.deg)
        self.assertTrue(small.is_acute())
        big = Angle(320)
        total = small.add(big)
        self.assertEqual(20, total.deg, 'Adding')

# running

      5.2 GS - Inversion

      J. Hadamard defined a problem ill-posed if not all of the following are true:

      • the solution exists
      • the solution is unique
      • the solution is stable

      Later on, A. N. Tikhonov introduced regularization to approximate an ill-posed problem by a number of well-posed problems:
      Assume \( (\vec{m}_{0}, \vec{d}_{0}) \in M X D \) with the properties

      • \( A_{\vec{s}}(\vec{m}_{0}) = \vec{d}_{0} \)
      • \( A_{\vec{s}}\) is approximately linear when close to \( \vec{m}_{0} \)

      If sensitivity \( S_{\vec{m}_{0}}\) is defined as the ratio of the norm of the pertubation to the norm of the pertubation of the model parameter, then the maximum sensitivity is the operator norm
      \( S^{\mbox{max}}_{\vec{m}_{0}} = ||A_{\vec{m}_{0}}|| \).

      If \( \delta \) is the error of observation, we may determine variations of the model if
      \begin{equation*} ||\vec{m} - \vec{m}_{0}|| \geq \delta / ||A_{\vec{m}_{0}}|| \end{equation*}

      The following are classes of inversion methods:
      • Lookup tables
      • Iterative deterministic (LMA, Gauss-Newton, Gradient Descent)
      • Iterative stochastic by the Bayesian theorem (MCMC, multichain sampling, nonparametric sampling)
      • Deep Neural

      5.3 GS - Rock Physics

      Classical elastic theory describe materials to first order in the deformation vector \( \vec{u}(\vec{x})\). Hooke's law connects the strain vector \( \sigma_{ij} \) to
      the stress tensor \( \epsilon_{ij} = \frac{1}{2}(\frac{\partial u_{i}}{\partial x_{j}} + \frac{\partial u_{j}}{\partial x_{i}}) \) via the stiffness tensor \( C_{ijkl}\): \begin{equation*} \shoveleft \sigma_{ij} = C_{ijkl}\epsilon_{kl}.\\ \end{equation*} Due to the symmetriess \( C_{ijkl} = C_{jikl} = C_{klij} \) , there are at most 21 independant component in stead of \( 3^{4} = 81\) components.

      5.3.1 GS - Voigt notation

      \begin{equation*} C = \begin{bmatrix} C_{11} & C_{12} & C_{13} & 0 & 0 & 0 \\ C_{21} & C_{22} & C_{23} & 0 & 0 & 0 \\ C_{31} & C_{32} & C_{33} & 0 & 0 & 0 \\ 0 & 0 & 0 & C_{44} & 0 & 0 \\ 0 & 0 & 0 & 0 & C_{55} & 0 \\ 0 & 0 & 0 & 0 & 0 & C_{66} \end{bmatrix} \end{equation*}

      6.2 M - Linear Algebra

      Let \begin{equation*} M = \begin{bmatrix} A & B \\ C & D \end{bmatrix} \end{equation*} and assume that submatrix A is invertible. If the matrix \( \Delta = D - CA^{-1}B \) is also invertivle, then we have \begin{equation*} M^{-1} = \begin{bmatrix} (A^{-1}+A^{-1}B\Delta^{-1}CA^{-1}) & -A^{-1}B\Delta^{-1} \\ -\Delta^{-1}CA^{-1} & \Delta^{-1} \end{bmatrix} \end{equation*}
